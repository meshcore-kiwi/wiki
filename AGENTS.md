# MeshCore NZ Wiki

Community wiki for MeshCore in New Zealand. Astro + Starlight + Six theme, hosted on Cloudflare Workers.

## Writing style (applies to all content, human or AI-written)

- **British/NZ English**: colour, organise, centre, licence (noun), metre, modelling. Never American spellings.
- **No em-dashes (—)**: use ` - ` instead.
- **Minimal emojis**: none in headings or prose. Functional markers only (e.g. a tick/cross comparison) where they carry meaning.
- Official MeshCore URLs only: meshcore.io, flasher.meshcore.io, app.meshcore.nz, docs.meshcore.io. **Never meshcore.co.uk.**

## Content

- Pages live in `src/content/docs/`. The sidebar is fully autogenerated from the folder structure - never add a `sidebar` config to `astro.config.mjs`.
- Folder groups take the lowest `sidebar.order` of their pages; unordered items sort alphabetically after ordered ones.

## Build

`npm run build` - CSS minification must stay on esbuild (lightningcss rejects a selector in the Six theme).

`worker-configuration.d.ts` is generated by `npm run cf-typegen` but IS committed on purpose - do not gitignore or delete it.

The worker name `meschore-kiwi-wiki` in wrangler.jsonc is intentional, not a typo - do not "fix" it.

## D1 database

The `wiki_users` D1 database must live in Oceania. The region is fixed at creation and cannot be set in wrangler.jsonc, so if it ever needs recreating, always use:

```sh
npx wrangler d1 create wiki_users --location oc
```

Then update `database_id` in wrangler.jsonc with the new ID.
